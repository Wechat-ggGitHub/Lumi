# Aiva Onboarding 重新设计

**日期**: 2026-05-16
**目标**: 极速到达「魔法时刻」（按右 Option 说话 → Claude 干活）
**流程**: Welcome → 辅助功能 → 火山引擎凭证 → Provider + Key → 完成（1.5s）→ 主界面

---

## 流程概览

```
┌─────────────────────────────────────────────────────────────┐
│ Welcome (无进度点)                                            │
├─────────────────────────────────────────────────────────────┤
│ 辅助功能 (●○○○)                                              │
├─────────────────────────────────────────────────────────────┤
│ 火山引擎凭证 (○●○○)                                          │
├─────────────────────────────────────────────────────────────┤
│ Provider + Key (○○●●)                                        │
├─────────────────────────────────────────────────────────────┤
│ 完成提示 (1.5s) → 主界面                                     │
└─────────────────────────────────────────────────────────────┘
```

**删除的元素**: 工作目录页（CWD）、Done 页（替换为 1.5s 过渡）

---

## 共用页面外壳

所有配置屏（辅助功能、火山引擎、Provider）共用：

- **顶部**: 左侧「← 返回」按钮，右侧 4 个进度点（仅指示，不可点击）
- **返回按钮**: Welcome 不显示，其余 3 屏显示
- **键盘**: 不处理 Enter/Esc，纯鼠标/触控操作

| 进度点 | 不显示 |

## 第 2 屏: 辅助功能权限

**进度**: ●○○○

```
┌────────────────────────────┐
│ ← 返回          ● ○ ○ ○    │
├────────────────────────────┤
│                            │
│           ⌥                │
│                            │
│    开启快捷键监听           │
│                            │
│ Aiva 需要用右 Option 键     │
│ 来唤起语音输入。            │
│                            │
│   ┌──────────────────┐     │
│   │ 打开系统设置      │     │
│   └──────────────────┘     │
│       已授权               │
│                            │
└────────────────────────────┘
```

| 元素 | 内容 |
|------|------|
| 图标 | ⌥ (48px 大小，居中显示) |
| 标题 | 开启快捷键监听 |
| 副标题 | Aiva 需要用右 Option 键来唤起语音输入。 |
| 主按钮 | 打开系统设置 |
| 次按钮 | 已授权 |
| 进度点 | 显示，当前第 1 个激活 |

**交互行为**:
- 用户点击「打开系统设置」后，按钮文案保持不变（方便用户重新打开）
- 如果用户进屏时已授权 → 自动跳过，直接进下一屏
- 用户关掉系统设置回到 Aiva 但未授权 → 静默处理，不做提示

## 第 3 屏: 火山引擎凭证

**进度**: ○●○○

```
┌────────────────────────────┐
│ ← 返回          ○ ● ○ ○    │
├────────────────────────────┤
│                            │
│  语音识别与合成配置         │
│                            │
│ Aiva 使用火山引擎进行语音   │
│ 识别和语音合成。请填写凭证。 │
│                            │
│ ┌──────────────────────┐  │
│ │ App ID               │  │
│ ├──────────────────────┤  │
│ │ Access Token         │  │
│ └──────────────────────┘  │
│                            │
│   ┌──────────────────┐     │
│   │ 保存并继续        │     │
│   └──────────────────┘     │
│   如何获取凭证？            │
│                            │
└────────────────────────────┘
```

| 元素 | 内容 |
|------|------|
| 标题 | 语音识别与合成配置 |
| 副标题 | Aiva 使用火山引擎进行语音识别和语音合成。请填写凭证。 |
| 输入框 1 | placeholder: "App ID" |
| 输入框 2 | placeholder: "Access Token", type="password" |
| 主按钮 | 保存并继续 |
| 教程链接 | 按钮下方「如何获取凭证？」(链接待补充) |
| 进度点 | 第 2 个激活 |

**状态**:
- 输入框为空时 → 主按钮禁用（半透明，不可点击）
- 验证中 → 按钮文案「保存中...」
- 验证失败 → 输入框下方红色文字「凭证验证失败，请检查后重试」

| 进度点 | 第 3、4 个激活 |

**交互**:
- 点击「更多服务商」→ 就地在下方展开其余选项（OpenRouter 等）
- 验证失败 → 红色文字「API Key 不正确」

## 文案一致性

| 术语 | 使用 |
|------|------|
| 右 Option | 全局统一，不用 ⌥ 符号 |
| Aiva | 自称用「我」，Aiva 这个名字只在标题出现 |
| 辅助功能权限 | 避免使用系统术语，用「快捷键监听」 |
| 火山引擎 | 明确点出平台名，建立信任 |
| 语音识别与合成 | 不说 TTS/ASR，用用户能理解的词 |
