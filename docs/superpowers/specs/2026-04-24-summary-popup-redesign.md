# 摘要面板 & 详情窗口重设计

日期：2026-04-24

## 目标

重新设计菜单栏摘要弹窗和详情窗口，使核心用户路径最短：**点击托盘图标 → 在小面板中直接看到最近任务的结论**。详情窗口升级为可交互的对话界面，支持查看完整执行过程（含工具调用）并继续对话。

## 设计原则

1. **结论优先** — 面板标题用任务摘要（非用户输入），输出结果尽可能完整展示
2. **场景感知** — 不同应用状态（空闲、执行中、失败、录音等）显示不同的信息结构
3. **渐进展开** — 面板看结论 → 展开历史看列表 → 点进详情看完整对话
4. **短路径** — 大多数场景一次点击即达目标信息

## 一、摘要面板（Summary Popup）

### 1.1 窗口规格

- 尺寸：380 × 480 px（当前 380 × 400，高度增加 80px 以容纳更多内容）
- 位置：托盘图标正下方，居中对齐，间距 4px
- 属性：无边框、always-on-top、blur 自动关闭、可切换开关
- 保持现有 toggle 行为（再次点击关闭）

### 1.2 布局结构（通用框架）

```
┌─────────────────────────────────┐
│ [状态点] 状态文字    元数据（右侧）│  ← 状态栏
│                                  │
│ 任务摘要标题（大字）              │  ← 标题区
│ "用户原始 prompt"（斜体灰色）     │
│                                  │
│ ┌──────────────────────────────┐ │
│ │                              │ │
│ │   内容区（根据状态变化）       │ │  ← 内容区（flex: 1）
│ │                              │ │
│ └──────────────────────────────┘ │
│                                  │
│ [查看完整结果 →]（仅内容溢出时）  │  ← 溢出引导
│ ─────────────────────────────── │
│ ▶ 历史记录 (N)                   │  ← 历史（可折叠）
└─────────────────────────────────┘
```

- 内容区 `flex: 1` 占据主要空间
- 内容区设 `overflow: hidden`，当文本超出时在底部叠加渐变遮罩（`linear-gradient(transparent, 背景色)`），并显示「查看完整结果 →」链接
- 「查看完整结果」点击后打开详情窗口
- 历史区默认收起（`▶ 历史记录 (N)`），点击展开

### 1.3 各状态内容

#### 状态 A：idle + 最近已完成（最常见）

- 状态栏：绿色点 + "已完成" + "耗时 · 轮数 · 费用"
- 标题：`summary` 字段的首行或截断（作为简短标题），无 summary 时用 `user_prompt` 截断
- 副标题：`user_prompt`（斜体灰色，引号包裹）
- 内容区：`summary` 字段全文（完整展示，超长时截断+渐变遮罩）
- 点击内容区或「查看完整结果」→ 打开详情窗口

#### 状态 B：idle + 最近任务失败

- 状态栏：红色点 + "执行失败" + "耗时 · 轮数"
- 标题：`summary` 字段（若有）或 `user_prompt` 前几个字
- 副标题：`user_prompt`
- 内容区：错误信息（红色边框背景），附已完成的步骤概要
- 背景色使用 `#2a1a1e`（微红）而非默认深色

#### 状态 C：executing（执行中）

- 状态栏：蓝色脉冲点 + "执行中" + "已用时 · 已执行轮数"
- 标题：`user_prompt`（执行中尚无 summary，用 prompt 作标题）
- 副标题：无
- 内容区：
  - 旋转 loading 图标 + 当前 SDK 子状态文字（"正在思考..."、"正在执行工具..."）
  - 已执行工具调用列表（每条显示：操作类型 + 文件路径，已完成打 ✓，当前高亮蓝色，未执行灰色）
  - 列表实时更新，跟随 SDK 子状态变化

#### 状态 D：recording / transcribing（录音/转写中）

- 状态栏：紫色脉冲点 + "录音中" / "转写中"
- 内容区居中：音频波形动画 + 提示文字
- 录音中："正在聆听..." + "松开按键结束录音"
- 转写中："正在转写语音..."

#### 状态 E：idle + 无历史（首次使用）

- 状态栏：灰色点 + "待命"
- 内容区居中：⌘ 图标 + "按右 Command 开始语音输入" + "说出你的指令，Aiva 会帮你执行"

#### 状态 F：editing（编辑指令）

- 状态栏：紫色点 + "编辑中"
- 标题区：显示当前编辑中的转写文本
- 内容区：提示用户可在 voice-bar 中编辑

### 1.4 历史记录区

- 默认收起：`▶ 历史记录 (N)`，N 为总记录数
- 点击展开后标题变为 `▼ 历史记录`，下方显示列表
- 列表最多显示 **10 条**（当前 5 条），按时间倒序
- 每条记录显示：
  - 摘要标题（`summary` 字段截断）
  - 耗时 · 轮数
  - 相对时间（"3分钟前"、"1小时前"、"昨天"）
  - 失败任务用红色文字标记
- 点击任意历史记录 → 打开该记录的详情窗口
- 展开后历史区域可滚动，不影响上方当前任务内容

## 二、详情窗口（Detail Window）

### 2.1 窗口规格

- 尺寸：500 × 600 px（当前 500 × 600 保持不变）
- 位置：屏幕居中
- 标题栏：使用系统原生标题栏，显示任务摘要
- 可调整大小

### 2.2 布局结构

```
┌──────────────────────────────────────┐
│ [状态点] 任务摘要标题    元数据（右侧）│  ← 标题栏
├──────────────────────────────────────┤
│                                      │
│  [用户消息气泡 — 右对齐]              │
│                                      │
│  Claude                              │
│  [Claude 回复气泡 — 左对齐]           │  ← 对话区（可滚动）
│                                      │
│  ✓ 读取文件  src/lib/auth.ts    ▶    │  ← 工具调用（默认收起）
│  ✓ 编辑文件  src/lib/auth.ts    ▼    │
│    ┌─ diff 内容（展开状态）──────┐    │
│    │ - import session ...        │    │
│    │ + import jwt ...            │    │
│    └─────────────────────────────┘    │
│                                      │
│  Claude                              │
│  [最终回复气泡]                       │
│                                      │
├──────────────────────────────────────┤
│ [输入框: 输入后续指令...]  [🎤语音]   │  ← 输入区
└──────────────────────────────────────┘
```

### 2.3 对话区

- 气泡式布局：用户消息右对齐（紫色背景），Claude 回复左对齐（深灰背景）
- Claude 回复前显示 "Claude" 标签（紫色小字）
- 消息内容支持 Markdown 渲染（代码块、列表、粗体等）
- 对话区自动滚动到底部（新消息到来时）
- 加载时从数据库恢复完整对话历史

### 2.4 工具调用

- 默认**收起**：显示一行，包含 ✓/✗ 状态图标、操作类型（"读取文件"、"编辑文件"、"运行命令"等）、目标文件/命令
- 收起状态右侧显示 `▶` 箭头
- 点击展开显示详细内容：
  - 文件操作：显示 diff（红色删除行、绿色新增行，等宽字体）
  - 命令执行：显示命令输出
- 展开后箭头变为 `▼`，再次点击收起
- 样式：深色背景 + 细边框，区别于普通消息气泡

### 2.5 输入区

- 底部固定，不随对话滚动
- 左侧文字输入框（圆角，深灰背景）
- 右侧语音按钮（紫色圆形，麦克风图标）
- 文字输入：回车发送，Shift+回车换行
- 语音按钮：点击开始录音，再次点击结束并发送
- 发送后复用 `sdk_session_id` 继续同一 Claude 会话
- 新的对话轮次实时追加到对话区

### 2.6 会话复用

- 通过 `sdk_session_id` 恢复 Claude 会话上下文
- 用户发送后续消息时，调用 `claude-client.ts` 的 `query()` 并传入已有 session ID
- 新的回复和工具调用实时流式显示在对话区
- 费用、耗时等元数据实时更新

## 三、数据变更

### 3.1 数据库 schema 变更

需要在 `execution_history` 表中存储完整对话消息，新增 `messages` 字段：

```sql
ALTER TABLE execution_history ADD COLUMN messages TEXT;  -- JSON array of messages
```

消息结构：
```typescript
interface ConversationMessage {
  role: 'user' | 'assistant';
  content: string;
  toolCalls?: ToolCallRecord[];
}

interface ToolCallRecord {
  type: 'read_file' | 'edit_file' | 'run_command' | 'write_file' | 'other';
  target: string;        // 文件路径或命令
  status: 'completed' | 'failed';
  detail?: string;       // diff 内容、命令输出等
}
```

- `messages` 字段在执行过程中逐步追加更新
- 打开详情窗口时从该字段恢复对话历史

### 3.2 IPC 消息变更

现有 IPC 通道调整：

| 通道 | 方向 | 变更 |
|------|------|------|
| `summary:update` | main → renderer | payload 增加 `state` 字段（当前 AppState），用于决定面板内容结构 |
| `summary:open-detail` | renderer → main | 不变，传入执行记录 ID |
| `summary:fetch-detail` | renderer → main | 不变，请求完整记录（含 messages） |

新增 IPC 通道：

| 通道 | 方向 | 用途 |
|------|------|------|
| `detail:send-message` | renderer → main | 在详情窗口发送后续消息（文字内容） |
| `detail:start-voice` | renderer → main | 在详情窗口启动语音输入 |
| `detail:voice-result` | main → renderer | 语音转写结果回传到详情窗口 |
| `detail:stream-chunk` | main → renderer | Claude 流式回复的增量内容 |
| `detail:tool-call` | main → renderer | 新的工具调用事件 |
| `detail:execution-complete` | main → renderer | 执行完成通知（含最终费用/耗时） |

## 四、实现范围

### Phase 1：摘要面板重设计（优先）

- 修改 `SummaryPanel.tsx`，按状态渲染不同内容结构
- 面板高度从 400px 调整为 480px（`summary-popup.ts`）
- 实现历史记录折叠/展开
- 实现内容溢出检测 + 渐变遮罩 + "查看完整结果"链接
- 修改 `summary:update` IPC payload，增加 `state` 字段

### Phase 2：详情窗口对话界面

- 新增数据库 `messages` 字段，修改 `db.ts` 读写方法
- 修改 `claude-client.ts`，在执行过程中捕获消息和工具调用，写入 `messages`
- 重写详情窗口页面（`src/app/summary/detail/page.tsx`），改为对话界面
- 实现工具调用收起/展开组件
- 实现底部输入区（文字 + 语音）

### Phase 3：会话复用

- 实现详情窗口发送后续消息的逻辑
- 通过 `sdk_session_id` 恢复 Claude 会话
- 流式显示新的回复和工具调用
- 实时更新费用和耗时元数据
